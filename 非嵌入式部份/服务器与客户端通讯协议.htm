基本定义

名词约定

名称

说明

控制盒

智能家居的中心设备，用于管理家居中的其它设备。控制盒接入互联网，与服务器直接通信

设备

智能家居中接入控制盒的其他工作设备，如插座、智能锁等

客户端

控制终端，如手机

控制盒唯一标识

字符串，用于服务器标记控制盒，使用控制盒操作系统提供的读唯一码方式得到控制盒唯一标识。

 

技术选择

服务器程序     ：使用Java，采用Apache Tomcat服务器。

数据库              ：使用MySQL数据库，单表千万级数据性能高，稳定。

通信协议         ：HTTP协议

加密方式         ：RC4加密方式，加密密钥为用户名+用户密码，服务器端不存储用户密码，可以确保任何控制命令必须在有事先输入密码的场合下使用。

协议数据包格式

使用HTTP的 POST方式

请求参数形式,使用标准HTTP请求方式，可传递字符或JSON对象

1.  cmd=xxx&param1=value1&param2=value2  //常规参数传递

2.  cmd=xxx&param1={“key1”:value1,”key2”:value2}  //json对象传递

 

响应数据形式使用JSON

客户端协议

客户端不需要长连接，有请求时连接服务器并交换数据，空间时只要定时请求服务器即可。

 

登录协议 cmd=login

客户端请求登录用户

 

请求数据：

名称

数据类型

说明

username

String

用户名

password

String

密码，使用ApiUtil.simpleEncrypt加密,controllerID参数为username

controllerId

String

主机唯一ID号

控制主机使用controllerID作为用户名，密码固定为372ab6cab20bfcc2c63411c7863adbf5，由于password是经过以上加密，所以不同控制器传输的password明文不同。

 

响应数据：

名称

数据类型

String

访问客户端接口的token

controllerID

String

该用户的控制盒标识符

int

控制器是否与服务器正常连接

天气ID，暂定

客户端长轮询cmd=poll

用于服务器推送数据到客户端

请求数据：

名称

数据类型

说明

accessToken

String

登录时获得的token

version

long

版本号，第一次poll不用带版本号，每次请求需要传入服务器返回的最新版本号

 

响应数据：

名称

数据类型

说明

status

int

轮询状态

1 -有数据更新，此时devices,crons,scenes,rooms有数据返回

0-连接时间太长，服务器主动断开，此时客户端应立即再次请求服务器。

version

long

版本号

devices

Array<Device>

最新的设备状态

rooms

Array<Room>

最新的房间列表

crons

Array<Cron>

最新的定时器列表

scenes

Array<Cron>

最新的场景列表

 

长轮询的实现步骤：

1. 初始化version=0

2. 请求cmd=poll

3. 等待服务器返回

如果服务器返回status=0 表示没有数据推送，跳到第2步

如果服务器返回status=1表示服务器有数据推送，使用服务器返回的数据更新，设置version变量为服务器返回的version值，完成后跳到第二步。

如果出现网络错误，跳到第二步
