智能家居系统

关于整个家庭的 初始化、注册、授权、注销、更改密码的逻辑说明

一  客户端术语解释

客户端：指的是一个软件，该软件可以获取家庭内的连网设备的状态，可以控制这些设备。前提是有足够的权限。客户端有很多种类，安卓版，IOS版，以后还会有PC版，网页版。

唯一ID号：每个主机，都会有一个唯一的ID号，该ID号为72位，9个字节。在主机内的Zigbee片中，有一个由片厂家提供的，保证全球唯一的64位ID号，高位再补8位0，则成为我们的72位唯一ID号。

          由于以后会有非Zigbee的节点加入本系统，为了避免与Zigbee的唯一ID号重复，所以整体的唯一ID号使用72位。

 

二  客户端安全使用概述

在日常使用中，客户端需要使用账号与密码登录才可使用。在客户端可以正常登录使用之前，需要注册与授权，关于注册与授权，下面会有详细说明。

三  初始化

说明：

在开始使用前，服务器上并未有该主机的任何数据。需要一个初始化的激活动作，使服务器建立整个家庭的数据。

实现算法：

在家庭路由器主机未激活前，服务器上没有任何关于该主机的数据，路由器上记录着本机未激活的状态。在家庭主机激活后，服务器上建立了关于该家庭的数据库。由于不同的家庭，

是以72位唯一ID号作区分。所以在激活时，需要使用72位ID号。为了安全起建，要对该传给服务器激活的72位ID号进行验证，验证方法为使用AES加密，

对72位ID号执行操作，生成与该72位ID号唯一对应的KEY，再使用该KEY对72位ID号进行加密，生成一组16字节的密文，服务器收到后，先对72位ID作同样的运算，

得出AES的KEY，再解密密文，如果得出来就是72位ID号，则验证成功。

 

初始化成功后，路由器的状态切换为已激活，这时才可以进行其他的操作。

 

结果：

服务器收到合法的72位ID后，会为这个家庭的建立数据库，不过内容暂时是空的，只是有了这个家庭的记录。

路由器的状态切换至已激活，可以进行注册，授权等操作。

 

四  注册

说明：

在初始化（激活）后，可以进行用户的注册动作。注册指的是用户注册一个新的用户名与密码，以后可以使用该用户名与密码来登录，去家庭设备进行控制。

实现算法：

使用Zigbee内的64位ID号，使用一个算法，生成一一对应的SN码，该码贴于智能家居系统的保修卡上。

客户端在内网启动，使用该 SN码 + 用户名 + 密码传给路由器程序，路由器程序确认 SN码、用户名、密码都合法后，偿试使用SN码 + 72位ID号登录公网服务器，

请求服务器为本家庭添加新账号，如成功收到服务回复，则回复客户端注册成功。如果连接服务器失败或服务器返回注册失败与失败原因，回复客户端注册失败。

同一个家庭内部，不可以有相同的用户名，不同的家庭可以有相同的用户名。

 

结果：

注册成功后，服务器会保存该家庭新增可登录的用户名与密码。路由器在回复客户端注册成功时，会同时把72位ID号传给客户端，客户端永久保存起来。

路由器把72位ID号传给客户端永久保存的过程，也叫做授权，下面会有说明。

 

 

五  授权

说明：

当客户由于某些原因，要在一台新手机上安装客户端软件，并开始使用，出于安全考虑，此时输入用户名与密码并不可成功登录。由于该手机未被授权。

此时只能在内网中，打开客户端，进入授权界面，输入 SN码 + 用户名 + 密码 来进行授权，在授权成功后，客户端才可正常登录。

如客户不在家庭内网中，而又需要授权的话，可以致电客服，申请临时授权。

 

实现算法：

客户端把 SN码 + 用户名 + 密码传给路由器，路由器对比 SN码正确，用户名的确存在，密码也正确，则授权成功，路由器把72位ID传给客户端保存下来。

临时授权办法如下，客服根据客户要求的临时授权，生成一个临时的72位ID，正常的72位ID，最后8位为0，而临时的ID最后8位为0x02，（0x01是WIFI开关的），

由管理软件把临时72位ID与该ID的有效时间写入服务器数据库，同时客服邮寄特殊apk给客户，客户运行该APK即写入72位临时ID到手机内。

客户在登录时，软件会自动找该临时的72位ID去登录服务器，当临时ID过时后，该客户端不可再登录，当该客户端回到内网中时，也不可以使用该临时ID登录内网，

这些，客户端应该在内网补回授权动作。实现可永久登录。

 

内网路由器，只可接受SN码 + 用户名 + 密码登录，而公网服务器，可接受 SN码 或 临时SN码登录。

 

结果：

经过授权的客户端，可以在内网或公网成功登录，未经授权的客户端，智能用户名与密码的情况下，不可登录，如果申请临时ID，可在公网登录，不可在内网登录，

未永久授权的客户端，回到内网后，应使用SN码进行授权。

整个授权的过程，服务器并不参与，服务器只检查登录者的SN码 + 账号 + 密码。

而临时授权时，需要把临时的SN码写入服务器。

六  注销

说明：

当一个账号不再使用时，可以执行注销动作。

 

实现算法：

客户端把 SN码 + 账号 + 密码传给路由器，可以实现注销动作。路由器收到客户端注册请求后，需要连接公网服务器，请求服务器删除用户，当收到服务器成功的应答后，

路由器回复客户端注销成功。

 

结果：

以后客户端不再可以使用该账号登录。

 

 

七  忘记密码时更改密码

说明：

当用户忘记密码时，可使用 SN码 + 账号 + 新密码传给路由器直接更改密码，当用户记得旧密码时，可不使用SN码，登录后再更改密码。

 

实现算法：

客户端把 SN码 + 账号 + 新密码传给路由器，可以实现忘记密码时的更改密码操作，路由器收到客户端请求后，需要连接公网服务器，当收到服务器更改密码成功的应答后，

路由器回复客户端更改密码成功。
