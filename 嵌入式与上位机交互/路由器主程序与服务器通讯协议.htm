路由器主程序与服务器通讯协议

一  概述

路由器主程序与服务器通讯时，主要工作内容有两个，一是接收所有客户端的请求改变设备状态的命令，二是把设备状态改变的命令发送给服务器。

 

二  数据模型定义

Account 数据模型

名称

数据类型

说明

username

String

用户名

passwork

String

密码

 

PhysicalDevice 数据模型

名称

数据类型

说明

devicePid

String

物理设备ID号，72位

devicePort

int

端口号，由于一个物理设备的一个72位唯一ID号会对应着多个逻辑设备，所以72位的唯一ID号未能达到逻辑设备与物理按钮（或其他可操作的部份）一一对应，加入port，使physicalId + port 是绝对唯一的组合，达到逻辑设备与物理按钮一一对应的目的。

deviceId

String

逻辑设备的ID号，有物理设备动作时，通过该ID号找到逻辑设备，改变逻辑设备的状态。

type

int

设备类型，参看 《逻辑设备的类型说明》

state

object

设备状态，参看 《逻辑设备的类型说明》

 

三  协议正文

激活协议 cmd=activate

客户端要求激活一个新的家庭

请求数据：

名称

数据类型

说明

controllerId

String

“0012345678abcdef00”,暂定为18位的16进制数的字符串

encodeId

String

“12345678901234567890123456789012” 32位十六进制数的加密内容，该32位16进制数的内容其实是 controllerId 经过一个 KEY 加密后的密文。

服务器使用该 KEY 对 encodeId 进行解密，则可以得出 controllerId ，从而确定本次命令的合法性。

 

而 KEY 并非所有用户相同，而是由 controllerId 根据一个算法产生，不同的 controllerId 将会产生不同的 KEY。同一 controllerId 则会产生同样的 KEY。

响应数据

名称

数据类型

说明

controllerId

String

18位唯一ID

result

int

激活结果

激活成功后，服务器开始建立该家庭的数据库

控制主机长轮询 cmd=cpoll

类似客户端长轮询，由控制主机向服务器发起，服务器向控制主机推送数据。推送的数据主要是客户端做的一些修改或客户端发送的命令。

 

请求数据：

名称

数据类型

说明

accessToken

String

登录时获得的token

version

long

版本号，第一次poll不用带版本号，每次请求需要传入服务器返回的最新版本号

 

响应数据：

名称

数据类型

说明

status

int

轮询状态

1 -有数据更新，此时crons,scenes,rooms,commands有数据返回

0-连接时间太长，服务器主动断开，此时客户端应立即再次请求服务器。

version

long

版本号

rooms

Array<Room>

最新的房间列表

crons

Array<Cron>

最新的定时器列表

scenes

Array<Cron>

最新的场景列表

controls

Array<ControlRequest>

客户端发送的控制请求，控制主机需要依次执行，并通过cmd=reply一次性向服务器发送控制结果

forceLogout

int

=0时正常，=1时客户端强制下线，重新回到登录界面，重新登录。

 

 

控制主机回复控制结果 cmd=reply

当服务器推送的cpoll中有控制命令时，执行完这些控制命令后需要调用该接口告诉服务器控制结果。

 

请求数据：

名称

数据类型

说明

accessToken

String

登录时获得的token

results

Array<ControlReply>

对应控制请求的结果,JSON字符串

 

响应数据：

名称

数据类型

说明

result

int

请求结果

1 -成功

0-失败，errorMsg有效

errorMsg

String

错误信息
